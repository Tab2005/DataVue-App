## 一、模組在解決什麼問題

平台自報的「哪個活動帶來最多轉換」是**最後點擊歸因**，會把常駐 always-on 活動、再行銷活動、品牌曝光活動的功勞**搶過來**。實際上這些活動多半是在收割已被其他活動「觸發」的需求，挪開它們未必真的掉轉換。

**MMM（Marketing Mix Modeling）從「各活動花費的時間序列共變」估計增量貢獻**，換言之它解的問題是：

> 如果把某個活動的預算砍掉（或加倍），**整體轉換大致會變動多少**？

讀圖時請記得這個口徑：橫軸單位是「真實增量轉換」而非「平台記在誰頭上的轉換」。

## 三、三聯對比圖的判讀

圖中每個組別有**三條橫條**（layout=vertical = 水平 bar），由上至下依序為：

- **Spend Share（花費占比）**——這個組吃掉了多少 % 的總預算
- **Platform-Reported（自報占比）**——Meta 後台記在這個組頭上的轉換占總轉換的 %
- **MMM Contribution（MMM 貢獻中位）**——模型估計這個組的**真實增量**貢獻占多少 %

三條都應該橫向加起來接近 100%（MMM 多出來的部分 = 基線貢獻，也就是「不投 Meta 也會發生的轉換」）。

### 3.1 三種典型型態

把三條 bar 從左到右比一比，可以歸到三種型態之一：

| 型態 | 視覺特徵 | 解讀 |
|---|---|---|
| **收割嫌疑** | 自報 >> MMM（自報遠高於 MMM） | 該組搶了別組的功勞。常見於：常駐 always-on 活動、brand 曝光、再行銷。**不該再加碼**，要評估是否要縮減 |
| **被低估** | MMM > 自報（MMM 遠高於自報） | 該組的真實貢獻被 Meta 漏記。常見於：隱私流失（iOS 14+ 後 cookieless 環境）、新客開發素材、跨裝置轉換。**值得加碼** |
| **健康歸因** | 自報 ≈ MMM | 該組的歸因大致誠實。預算決策可照平台 ROAS 走 |

> 數值差距多少算「遠」？依 MMM 驗證經驗：**絕對差距 > 5pp 值得關注，> 10pp 強烈訊號**（pp = percentage point）。但請同時看診斷卡（§ 五）確認資料可信。

### 3.2 真實帳號案例（act_1077213689343030）

驗證文件 0.3 節的數字翻成圖像後：

| 組別 | 花費 | 自報 | MMM（中位） | 型態 |
|---|---|---|---|---|
| G1 主力常態（always-on） | 46.9% | 49.1% | **29.3%** | 收割嫌疑（自報 49 vs 真實 29，差 20pp） |
| G2 主力影片 | 13.1% | 13.5% | 10.8% | 接近健康 |
| G3 官網檔期 | 13.1% | 12.4% | 16.2% | 略被低估（檔期拉新） |
| G4 社群自投 | 9.5% | 11.1% | **16.6%** | 強烈被低估（自投素材被低估是常見） |
| G5 大包裝 | 7.2% | 6.6% | **11.2%** | 強烈被低估 |
| G6 小額常態 | 7.0% | 6.4% | 0% | **存疑**（與其他組負相關，共線假象，見 § 五） |
| G7 測試導流 | 3.1% | 1.0% | 0% | **存疑**（資料量不足，見 § 五） |

**決策結論**：把 G1 部分預算挪到 G4 / G5，預期整體增量轉換會上升（邊際報酬見 § 四）。G6 / G7 不要依此結果做任何決策。

### 3.3 表格的補充欄位

圖下表格除了重畫三條 bar 的數字外，還多了：

- **MMM (median / min / max)**：中位數是**點估計**，min–max 是 5 次重啟的**區間**。**永遠帶著區間讀**：min 與 max 差 10pp 以上的組別，結論要打折扣（可能因共線、資料量而搖擺）
- **Marginal / +N 元**：每投入 +N 元帶來的邊際轉換，**與下一節的排序圖同步**

### 3.4 視覺陷阱提醒

- **三條 bar 是「占比」不是「絕對轉換數」**：G1 即使 29% 貢獻仍可能貢獻最多絕對數。決策時別只看占比，要回頭乘上總轉換
- **基線（不投也會發生的轉換）**會吃掉一部分百分比：MMM 全部加起來不是 100%，剩下的就是基線。基線占比也是訊號（基線太高 = 預算應縮減、把錢省下來）
- **同色 = 同語義**：同組的三條 bar 採同一組色（藍/綠/黃），跨組同色一致。**不要看到「同色」就誤判是同活動**

## 四、邊際報酬排序圖的判讀

### 4.1 圖本身

- 單 series 橫向 bar 圖，由上至下**從高到低**排序
- X 軸單位：**每 +N 元投入帶來的增量轉換數**（N = `marginal.step`，由系統依帳戶幣別/花費量級自動選；常見 100、300、500；下方副標明示）
- 最高的那條 bar 右端有「+0.XX 最佳」標籤（直接端點標籤，per dataviz 規則只標極端一條）
- 「存疑」組同樣以灰階呈現

### 4.2 怎麼讀

**直接回答「下一塊錢該投給誰」**：

- 第一名 = 邊際報酬最高的組 = 預算應**加碼**的第一順位
- 倒數第一 = 邊際報酬最低的組 = 預算應**減碼**的第一順位
- 排序差距不大（例如前 3 組都在 ±10% 內）= 邊際上接近，**加碼優先選最有成長空間的那組**（通常是自報遠低於 MMM 的被低估組）
- 倒數一兩名若也接近 = 對整體貢獻有限，**不一定要砍**（避免動到穩定收割）

### 4.3 真實案例接續 § 3.2

| 組別 | 邊際 / +100 元 | 排序決策 |
|---|---|---|
| G1 主力常態 | 0.164 | 中段，不優先動 |
| G2 主力影片 | 0.168 | 中段 |
| G3 官網檔期 | 0.165 | 中段 |
| G4 社群自投 | **0.300** | **加碼首選** |
| G5 大包裝 | **0.358** | **加碼次選** |
| G6 / G7 | — | 存疑，不採信 |

### 4.4 邊際報酬的限制

副標有註明，**這是局部斜率**：

- 只在**目前花費水位附近**有效；G5 從日花費 7% 拉到 15%，邊際會掉（hill saturation）
- **不可線性外推**到「把整個預算都丟過來會怎樣」
- 對存疑組（G6/G7）不適用，因為共線假象下的邊際沒意義

要更全面的「如果整體重分配」情境，留待 docs/21 第 3 波 B 的預算最佳化任務（尚未立案）。

## 五、診斷卡 / 存疑狀態的判讀

診斷卡是這個模組**最重要的部分**，比主圖更值得讀。MMM 估計是**統計模型**，不是真理，診斷卡告訴你這次估計的可信度。

### 5.1 R² 與 Poisson 雜訊天花板

- **Holdout R² (median)**：模型在「沒看過的末段資料」上能解釋多少變異。**0.1 以上堪用，0.3 以上算好**。驗證案例實帳號 = 0.16（偏低但符合電商高雜訊特性）
- **Poisson Ceiling (holdout)**：日轉換服從 Poisson 分布下，**純雜訊造成的 R² 上限**。Holdout R² 接近天花板 = 已經把可解釋的訊號榨乾；遠低於天花板 = 還有沒被模型吃進去的結構（例如季節性、假日衝擊、競品動態）
- **比例原則**：Holdout R² / Poisson Ceiling **> 50%** 表示模型在合理範圍內，**< 30%** 表示估計有雜訊，**結論要保守**

> 真實案例：Holdout R² 0.16 / 天花板 0.58 → 利用率 28%。數字是「可用但有雜訊」級距，**任何決策請保守**。

### 5.2 共線性警告

**這是模型的最大敵人**。如果兩個組的日花費高度相關（r > 0.7），模型就分不清「轉換是誰帶來的」。警告列出的 pair 會附 r 值。

- **r > 0.85** = 嚴重共線，估計幾乎不可信
- **r 0.7–0.85** = 中度共線，估計區間會被拉寬

**處置**：

1. **錯開預算調整**：把兩個組的預算調整時間錯開（例如 G1 週一動、G5 週三動），2–4 週後模型就能拆開它們
2. **改分組**：把其中一個組併入其他組（用分組編輯器），降低組數直到共線解除
3. **接受不準**：如果無法錯開也不願併組，請把共線組的貢獻視為「這兩個組合計的貢獻」而非各自分解

### 5.3 資料量檢查

- **Days of data**：分析期間天數。< 90 = 拒絕分析（資料太短）；90–180 = 堪用；180+ = 良好
- **Mean daily conversions**：日均轉換數。< 5 = 拒絕分析（雜訊淹沒訊號）

若 guardrail 預檢失敗，畫面會直接顯示原因，**不要硬跑**——結果會是噪音。

### 5.4 「存疑」標記

圖中**灰階 + DOUBTFUL 徽章**的組別 = 同時滿足兩個條件：

1. MMM 貢獻中位 ≤ 0.5%（幾乎被判 0）
2. 出現在共線性警告的 pair 裡

**對存疑組：不要做任何預算決策**。它的 0% 不是「這組真的沒用」，多半是「模型分不出功勞給誰」。

真實案例的 G6（與所有組負相關 r -0.6 ~ -0.67）就是典型：花費曲線和別人相反，模型當作「別人多花的時候它少花」就歸 0。**實際上 G6 可能是穩定的小額支撐活動，砍掉風險大**。

## 六、預算決策工作流（從讀圖到行動）

把上面整合成一個可重複的 SOP：

1. **看診斷卡**（先看 R² 與共線）——Holdout R² < 0.1 或共線警告一堆？回到步驟 0 調整資料/分組，不急著下結論
2. **看三聯對比，找出明顯訊號**——只挑「自報 vs MMM 差距 > 10pp」的組列入候選。差距 < 5pp 視為「歸因大致誠實」，照平台 ROAS 走即可
3. **看邊際排序，加碼/減碼順位**——從第一名往後排，扣掉存疑組
4. **核對區間寬度**——中位 vs min/max 差 10pp 以上？先標記「待重跑」，別當下大動
5. **制定行動**（一次只動一塊）：
   - 加碼：選邊際最高的 1–2 組，幅度 20–30%（hill 飽和曲線的合理線性區）
   - 減碼：選自報遠高於 MMM 的 1 組（典型 = G1 always-on），幅度對等
   - 錯開時間：加碼組與減碼組**不同週**執行，2–4 週後重跑分析驗證
6. **4 週後重跑 + 對照歷史快照**——若 MMM 估計的 G1 占比下降、新組占比上升，模型確認你的決策；反之回頭檢討

### 6.1 反例：千萬別這樣做

- ❌ **同時大砍 3 個組**：你會失去 baseline，4 週後模型無 baseline 可估，新結果會更不準
- ❌ **看到「邊際最高」就全押**：hill 飽和曲線，加碼到一定量後邊際會掉；保守分批
- ❌ **忽略共線警告直接重分配**：若 G1 × G5 共線，你動 G1 砍預算的「效果」會被模型誤算到 G5，反而更亂
- ❌ **依單次分析立刻決策**：跑 2–3 次（不同 seed）看區間穩定度，穩定再動
- ❌ **把「存疑」組的 0% 解讀為「無用」**：很可能是有用但被共線遮蔽，**不要砍**

## 七、常見案例速查

| 你看到的圖像 | 對應處置 |
|---|---|
| 自報 >> MMM（差 10pp+，組為 always-on / 再行銷） | 該組**減碼候選**；先錯開其他組預算再驗證 |
| MMM > 自報（差 10pp+，組為新客 / 自製素材） | 該組**加碼候選**；幅度 20–30%，4 週後重跑驗證 |
| 三條 bar 接近重疊 | 歸因誠實，**照平台 ROAS 決策即可**，本模組無特別建議 |
| 出現灰階 + DOUBTFUL 徽章的組 | **不要決策**；先併組或錯開預算重跑 |
| Holdout R² 遠低於 Poisson 天花板（< 30%） | 資料/分組需調整；**暫緩決策** |
| 邊際排序第一名的組佔比已經很高（> 30%） | 注意 hill 飽和；加碼幅度縮到 10–15% |
| 邊際排序倒數第一名 = 收割嫌疑組 | 雙訊號吻合，**減碼**；但要錯開時間 |
| 邊際排序倒數第一名 = 健康歸因組 | 該組已飽和或體質小，**保留不要動** |
| 全部組都被判 0%，基線 100% | 資料或分組有問題，**回到 § 五.5** |

## 八、其他注意事項

### 8.1 邊際步長 N 的解讀

`+100 元 帶來 +0.16 轉換` 跟 `+1000 元 帶來 +1.6 轉換` 是**等價敘述**（hill 飽和區間內可線性近似）。**N 是步長不是天花板**——你加碼 1,000 元不會得到 1.6 轉換，會略少（飽和效應）。

### 8.2 為什麼是「中位 + 區間」不是「單點」

模型用 5 次不同起始條件（seed）重跑，估計會落在某個範圍。**單純相信中位數是危險的**，要看區間寬度：

- min–max < 5pp = 模型對這組很有把握
- 5–10pp = 有一點不確定，決策要保守
- \> 10pp = 高度不確定，**不該基於這組決策**

### 8.3 歷史快照比對

頁面最下方的「歷史快照」可載入過去分析。比較兩次結果時關注：

- 某組的 MMM 占比是否朝你決策的方向移動（例如你減了 G1，G1 的 MMM 占比下降）
- 邊際排序是否洗牌（例如你加碼 G4 後，G4 邊際下降 = 飽和點到了）
- R² 是否改善（錯開預算後共線降低 → R² 上升）

### 8.4 幣別與口徑

當前快照的 `marginal_step` 由系統依帳戶幣別/花費量級自動選（預設 = 日均花費 1% 向上取整至 100 整數倍，下限 100）。不同帳戶可能得到不同的 N，**跨帳戶比較時要把 N 統一換算**（例如都換算成「+1000 元的邊際」再比）。

### 8.5 與 GA4 雙 y 對照（尚未上線）

docs/21 第 3 波 A 將新增「同帳戶 + 同期間」跑 meta-y 與 ga4-y 兩次，**差距即歸因通膨指數**：

- meta-y 高 / ga4-y 低 = 收割嫌疑（已被上線的「三聯對比」型態捕捉）
- ga4-y 高 / meta-y 低 = 被 Meta 低報（隱私流失、跨裝置）

兩種 y 計數口徑不同（去重/時區），**只比較貢獻占比與邊際排序，不比較絕對筆數**。上線後本指南會補 § 三.5。

## 九、限制與邊界

這套模型是**確定性統計計算**，不是機器學習；給的是估計不是預測。請勿：

1. **外推**：模型用最近 180 天資料，1 年後的市場結構（競品、消費者行為、平台演算法）可能已變
2. **跨帳號合併**：單帳號分析，合併多帳號會引入跨帳號共線問題
3. **跨平台合併**：目前只支援 Meta；Google Ads / LINE 投放需待 docs/21 第 3 波 B 的跨渠道擴充
4. **拿來當「是否上線某活動」的決策依據**：MMM 估的是「已上線活動的真實貢獻」，**新活動的潛力是 Andromeda 的素材評分（docs/12）負責**——兩者互補不互替
