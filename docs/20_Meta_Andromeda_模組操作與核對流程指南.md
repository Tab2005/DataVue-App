# 20 Meta Andromeda 模組操作與核對流程指南

本指南提供 `Meta Andromeda` 模組的**端到端完整操作流**，協助您逐一核對 Prediction、Observation、Learning 與 Release 各階段功能。

---

## 流程一：素材預估與審核流 (Pre-launch Prediction Flow)
* **目的**：驗證在素材投放前，系統能否進行正確評估、填寫回饋並進入審核佇列。

### 步驟與核對項：
1. **進入「評分工作台」**：
   * 點擊左側選單的 **「評分工作台」** (`/meta-andromeda/score-lab`)。
   * **核對項**：頁面正常載入，並顯示素材上傳拖曳區與評估參數輸入框（如 objective, market, primary_text 等）。
2. **上傳素材並提交**：
   * 上傳一張圖片或輸入測試素材的 `asset_uri`，輸入其他選填項目後，點擊 **「提交評估」**。
   * **核對項**：系統回傳評估狀態並將任務送入 queue；完成後顯示預估結果，包含 **預估 ROAS 區間 (low/mid/high)**、綜合評分、風險標籤與文本診斷。
3. **進入「審核佇列」填寫回饋**：
   * 點擊左側選單的 **「審核佇列」** (`/meta-andromeda/review-queue`)。
   * **核對項**：可以在列表中找到剛剛提交的素材，其狀態應為 `completed`。
   * 點選該素材進入細節頁面，在底部填寫備註或提交 Reviewer 審查回饋。
   * **核對項**：回饋成功寫入，且時間軸 (Timeline) 正常更新。

### 附錄：評分工作台欄位說明與填寫建議

評分工作台的核心是上傳**圖片或影片素材**，但表單中的文案與情境欄位對評分結果有直接影響，非裝飾性欄位。

#### 欄位與圖片評分的關係

| 欄位 | 對評分的影響 | 重要程度 |
|---|---|---|
| **廣告標題 (Headline)** | AI 評估圖片與標題的語意一致性（`relevance` 維度）；啟發式：有值 +8 分 | ★★★ 重要 |
| **廣告主要文字 (Primary Text)** | 影響 `copywriting` 維度評分；啟發式：有值 +8 分 | ★★★ 重要 |
| **行動呼籲 (CTA)** | 評估圖片內視覺 CTA 與文字 CTA 的一致性（`cta_clarity` 維度）；啟發式：有值 +10 分 | ★★★ 重要 |
| **行銷目標 (Objective)** | 調整評分嚴格度（purchase 目標對 CTA 清晰度要求最高）；啟發式：purchase +3 分 | ★★ 有效 |
| **版位系列 (Placement Family)** | 調整構圖評分基準（直式 Reels 與橫式 Feed 標準不同）；啟發式：適合版位 +4 分 | ★★ 有效 |
| **目標市場 (Market)** | 目前 AI 模型利用率偏低，主要作為 metadata 記錄 | ★ 低 |

#### 核心概念

文案欄位（Headline / Primary Text / CTA）**不是對圖片內文字的重複輸入**，而是讓 AI 能夠評估圖片與廣告文案的**一致性**。例如：
- 圖片主色為冷色系，但標題強調「暖心限定」→ 相關性（relevance）分數低
- 圖片有明顯購買按鈕，CTA 填「了解更多」→ CTA 一致性分數低

若文案欄位留空，AI 無法評估這些維度，整體評分準確性會下降，`copywriting` 和 `relevance` 兩個維度等同盲評。

#### AI 評分 vs 啟發式評分的欄位利用差異

**AI 評分**（`scoring_mode: ai`）：Prompt 中包含所有欄位值，模型綜合圖片視覺與文案語意評分。`diagnostic_breakdown` 的四個維度（`visual_appeal` / `copywriting` / `cta_clarity` / `relevance`）均被評估。

**啟發式評分**（`scoring_mode: heuristic`）：僅檢查欄位是否有值（純計分制，不分析內容），與圖片本身無關。基礎分 56（圖片）/ 52（影片），有 Headline +8、Primary Text +8、CTA +10、purchase +3、適合版位 +4，上限 88 分。

## 流程二：投放觀測與數據匯入流 (Post-launch Observation Flow)
* **目的**：驗證實際投放後，如何將 FB Ads 實體表現數據回灌至 Andromeda 作為觀測依據。

### 步驟與核對項：
1. **進入「成效分析」**：
   * 點擊左側選單的 **「成效分析」** (`/analytics`)。
   * **核對項**：頁面載入 Facebook Ads 表格。確保右側有出現固定的 **「操作」** 欄位。
2. **手動單筆/批次匯入**：
   * 找到具有 `ad_id` 且已投放的廣告列：
     * **單筆匯入**：點選該列最右側的 **「送出」** 按鈕。
     * **批次匯入**：勾選左側的「批次」核取方塊（選取多筆），點擊表格上方的 **「批次送出」** 按鈕。
   * **核對項**：列內狀態 badge 依序顯示 `未送出` -> `匯入中` -> `已送出` (綠色標籤)。
3. **快速篩選核對**：
   * 回到「成效分析」頁面，下拉上方工具列的 **「全部匯入狀態」** 篩選器，選擇 `已送出` 或 `未送出`。
   * **核對項**：表格內容正確進行過濾，僅顯示對應狀態的廣告。

## 流程三：漂移診斷對照與排除流 (Drift Diagnostics Flow)
* **目的**：驗證如何透過真實配對評估模型準確性，並排除異常干擾數據。

### 步驟與核對項：
1. **進入「監控總覽」**：
   * 點擊左側選單的 **「監控總覽」** (`/meta-andromeda/monitoring`)。
2. **手動觸發漂移檢查**：
   * 往下滑動頁面至 **「漂移檢查 (Drift Trigger)」** 面板。
   * 選擇您剛剛在流程二匯入廣告時對應的窗口（如 `last_24h`），點擊 **「執行漂移檢查」**。
   * **核對項**：頁面載入並產生一筆最新的 Drift Report。卡片上顯示真實的 `Accuracy` 與 `MAE` 指標。
3. **開啟診斷工作台**：
   * 點擊卡片下方的 **「查看診斷明細 (View Diagnostic Details)」** 按鈕。
   * **核對項**：右側成功滑出毛玻璃 Slide-over 面板，呈現 **預估與實際績效對比** 列表，各列顯示廣告名稱、預期 ROAS Band vs 實際 ROAS Band，以及偏差值。
4. **排除噪聲數據**：
   * 在明細列表中，點選某筆預估失準广告旁的 **「排除」** 按鈕。
   * **核對項**：該列變為半透明虛線框，且不會被計入頂部的打包校準清單。點選「加入」可重新恢復。

## 流程四：資料校準同步與安全鎖定流 (Calibration & Release Lock Flow)
* **目的**：驗證失準素材如何打包同步為校準集，以及漂移時版本發佈是否會被安全鎖定。

### 步驟與核對項：
1. **打包同步校準資料集**：
   * 在右側滑出面板的頂部，點擊 **「打包同步校準資料集」**。
   * **核對項**：系統跳出成功同步提示，將偏差素材的 `calibration` 狀態標記為 True 並保存於後端，用於下一輪模型微調。
2. **模擬漂移鎖定（發佈閘門驗證）**：
   * 若您手動觸發的漂移檢查顯示線上模型處於 **`drifted` (嚴重漂移)** 狀態（準確率 $< 60\%$ 或 $\text{MAE} > 0.50$）：
     * 點擊左側選單的 **「版本總覽」** (`/meta-andromeda/release`)。
     * **核對項**：
       1. 頁面左側顯示 **「線上實測對照證據」** 面板，且報告狀態為紅色的 **`DRIFTED`**。
       2. 右側的「候選版本」列表中，**「批准」** 按鈕處於禁用（置灰 disabled）狀態。
       3. 按鈕下方顯示紅色的發佈安全警告：
          > ⚠️ 線上模型已檢測出顯著漂移 (Accuracy: xx.x% < 60%)。為了避免劣質預估，已自動鎖定發佈。請先進入監控工作台執行「資料校準」，再行核准新模型。
