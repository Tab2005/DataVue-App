## M-4：清理 dependencies.py 的 logging.basicConfig

### 修改檔案

**`backend/dependencies.py`**

### 變更內容

```diff
- # Configure Logger
- logging.basicConfig(level=logging.INFO)
- logger = logging.getLogger(__name__)
+ # Configure Logger（basicConfig 由 core/logging.py 的 setup_logging() 統一設定，勿在此呼叫）
+ logger = logging.getLogger(__name__)
```

### 說明

`logging.basicConfig()` 在 `core/logging.py` 的 `setup_logging()` 之前執行時，
會於根 Logger 上掛載未格式化的 `StreamHandler`，導致日誌格式不一致（缺少時間戳、模組名稱等）。
移除後，所有日誌格式統一由 `main.py` 啟動時呼叫的 `setup_logging()` 管理。

## M-1：routers/auth.py 整合 integration_service

### 問題根因

`get_token_status` 端點直接讀取 `User` 表的廢棄欄位（`fb_access_token`、`token_expires_at`）。
`UserIntegration` 遷移完成後，這些欄位不再更新，導致 `token_exists` 永遠為 `False`。

### 修改檔案

**`backend/routers/auth.py`**

### 變更內容

**1. 新增 import**：

```python
from services.integration_service import (
    get_user_integration,
    get_decrypted_access_token,
)
```

**2. 重寫 `get_token_status` 端點**：

舊版（讀取 User 表廢棄欄位）：
```python
target = db.query(User).filter(User.google_id == user_id).first()
token_exists = False
if target and hasattr(target, 'fb_access_token') and target.fb_access_token:
    token_exists = len(str(target.fb_access_token)) > 10
```

新版（查詢 `user_integrations` 表）：
```python
integration = get_user_integration(db, user_id, "facebook")

token_exists = False
expires_at = None
is_expired = True
days_remaining = None

if integration and integration.access_token:
    token_exists = True
    expires_at = integration.token_expiry
    if expires_at:
        if expires_at.tzinfo is None:
            expires_at = expires_at.replace(tzinfo=timezone.utc)
        now = datetime.now(timezone.utc)
        delta = expires_at - now
        days_remaining = delta.days
        is_expired = days_remaining < 0
    else:
        is_expired = False  # 無過期時間視為有效
```

**3. 回傳格式更新**（新增 `provider` 欄位）：

```python
return {
    "token_exists": token_exists,
    "expires_at": expires_at.isoformat() if expires_at else None,
    "days_remaining": days_remaining,
    "is_expired": is_expired,
    "provider": "facebook",
}
```

> **注意**：`team_id` 參數保留於 API 簽章以維持 API 相容性，
> 未來可擴展支援 Team-level integration 查詢。

### 同步更新：tests/test_auth.py

新增 3 個單元測試驗證新邏輯：

| 測試名稱 | 描述 |
|---------|------|
| `test_token_status_no_integration_returns_false` | 無整合記錄時 `token_exists` 應為 `False` |
| `test_token_status_with_valid_integration_returns_true` | 有效 Token 且未過期，`token_exists=True`、`is_expired=False` |
| `test_token_status_with_expired_integration` | Token 已過期，`is_expired=True`、`days_remaining < 0` |

測試使用 `unittest.mock.patch` 隔離：
- `routers.auth.verify_google_token_and_get_sub`（跳過 Google Token 驗證）
- `routers.auth.get_user_integration`（模擬 DB 查詢）

## L-3：ga4_service.py 更新 cache API

### 問題根因

`ga4_service.py` 使用舊版快取函式（`get_cached`、`set_cached`、`get_cached_redis`、`set_cached_redis`），
語意已改變且邏輯重複（兩套 if/fallback 結構分別處理 Redis 和 L1）。

### 修改檔案

**`backend/ga4_service.py`**

### 變更內容

**1. 更新 Import（第 28–29 行）**：

```diff
- from cache import generate_cache_key, get_cached, set_cached, analytics_cache
- from redis_cache import get_cached_redis, set_cached_redis
+ from cache import generate_cache_key, cache_get, cache_set, analytics_cache
```

**2. 合併 Cache 讀取邏輯（舊為兩段 if-use_redis + L1 fallback，共 35+ 行）**：

舊版（兩段結構）：
```python
if use_redis:
    cached_full = get_cached_redis(base_cache_key)   # Redis 查詢
    ...
# L1 fallback
cached_full = get_cached(analytics_cache, base_cache_key)  # L1 查詢
```

新版（統一呼叫 `cache_get`，自動 L2→L1 fallback）：
```python
# 單一路徑，cache_get 自動處理 L1+L2 雙層快取、Redis 不可用時自動降級
cached_full = cache_get(base_cache_key)
```

**3. 簡化 Cache 寫入邏輯（舊為 redis_set boolean + 兩段 if，共 18 行）**：

舊版：
```python
redis_set = False
if use_redis:
    redis_set = set_cached_redis(key, result, redis_ttl)
if not use_redis or not redis_set:
    set_cached(analytics_cache, key, result)
```

新版（移除 `redis_set` boolean，統一 TTL 計算）：
```python
_cache_ttl = redis_ttl if use_redis else int(analytics_cache.ttl)
cache_set(key, result, _cache_ttl)
```

### 語意等同性確認

| 舊版呼叫 | 新版呼叫 | 語意等同 |
|---------|---------|---------|
| `get_cached_redis(key)` | `cache_get(key)` | ✅（cache_get 包含 L2 Redis 查詢） |
| `get_cached(analytics_cache, key)` | `cache_get(key)` | ✅（get_cached 內部已呼叫 cache_get） |
| `set_cached_redis(key, val, ttl)` | `cache_set(key, val, ttl)` | ✅（cache_set 寫入 L2+L1） |
| `set_cached(analytics_cache, key, val)` | `cache_set(key, val, 120)` | ✅（analytics_cache.ttl = 120s） |
