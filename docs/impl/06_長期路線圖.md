## 1. CI/CD：pytest 進入 PR Merge 門檻

### 目標

每次 Pull Request 合併前，自動執行 pytest 測試套件，覆蓋率未達門檻時阻擋 merge。

### 實作方案（GitHub Actions）

**檔案**：`.github/workflows/backend-tests.yml`

```yaml
name: Backend Tests

on:
  pull_request:
    branches: [main, develop]
    paths:
      - "backend/**"
  push:
    branches: [main]
    paths:
      - "backend/**"

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379
        options: >-
          --health-cmd "redis-cli ping"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python 3.12
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: "pip"
          cache-dependency-path: "backend/requirements*.txt"

      - name: Install dependencies
        working-directory: backend
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install -r requirements-dev.txt

      - name: Run tests with coverage
        working-directory: backend
        env:
          DATABASE_URL: "sqlite:///:memory:"
          REDIS_URL: "redis://localhost:6379/0"
          SECRET_KEY: "test-secret-key-for-ci"
          FERNET_KEY: "dGVzdC1mZXJuZXQta2V5LTMyLWJ5dGVzLWxvbmch"  # CI 用測試金鑰
          GOOGLE_CLIENT_ID: "test-client-id.apps.googleusercontent.com"
          DEBUG_MODE: "true"
        run: |
          pytest tests/ \
            --cov=. \
            --cov-report=xml \
            --cov-report=term-missing \
            --cov-fail-under=30 \
            -v \
            --tb=short

      - name: Upload coverage report
        uses: codecov/codecov-action@v4
        if: always()
        with:
          file: backend/coverage.xml
          flags: backend
          fail_ci_if_error: false

      - name: Comment coverage on PR
        uses: py-cov-action/python-coverage-comment-action@v3
        if: github.event_name == 'pull_request'
        with:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          COVERAGE_DATA_FILE: backend/.coverage
```

### 覆蓋率門檻計畫

| 時間點 | 最低門檻 | 說明 |
|--------|---------|------|
| 第一個月 | 30% | 基本 smoke tests 通過 |
| 第二個月 | 50% | CRUD 端點全面覆蓋 |
| 第三個月 | 65% | 整合測試（GA4/GSC mock）涵蓋 |

## 3. 前端：Google OAuth Token 靜默刷新

### 現狀

`frontend/src/utils/auth.js` 第 45 行有 TODO：

```javascript
// TODO: 實作 Google OAuth 靜默刷新（implicit flow）
// 目前：Token 到期時強制重新登入
```

`frontend/src/hooks/useTokenRefresh.js` 每 60 秒檢查 Token，到期時呼叫 `onExpired`（強制登出）。

### 技術方案分析

| 方案 | 優點 | 缺點 |
|------|------|------|
| Google One Tap 靜默登入 | 使用者無感知刷新 | 需要 HTTPS，第三方 Cookie 限制 |
| Authorization Code + PKCE | 最安全，無 Cookie 問題 | 需要後端配合 `/auth/refresh` 端點 |
| iframe 靜默刷新（已廢棄） | 歷史方案，不再推薦 | Safari/Firefox 封鎖第三方 Cookie |

**推薦方案**：Authorization Code + PKCE 流程

### 實作規劃

#### 後端新增 `/api/auth/refresh` 端點

```python
# backend/routers/auth.py 新增
@router.post("/refresh")
@limiter.limit("10/minute")
def refresh_access_token(
    request: Request,
    refresh_token: str = Body(..., embed=True),
    db: Session = Depends(get_db),
):
    """
    使用 refresh_token 換取新的 access_token。
    需要後端持有 refresh_token（存放於 user_integrations 表）。
    """
    # 1. 驗證 refresh_token
    # 2. 呼叫 Google OAuth2 TokenInfo API
    # 3. 回傳新的 access_token 和 expiry
    ...
```

#### 前端 useTokenRefresh.js 更新

```javascript
// frontend/src/hooks/useTokenRefresh.js
const tryRefreshToken = async () => {
  try {
    const refreshToken = getStoredRefreshToken();
    if (!refreshToken) {
      onExpired();  // 無法靜默刷新，強制登出
      return;
    }

    const response = await apiClient.post('/api/auth/refresh', {
      refresh_token: refreshToken,
    });

    if (response.access_token) {
      saveAuthToken(response.access_token);
      // 靜默刷新成功，使用者無感知
    }
  } catch (error) {
    onExpired();  // 刷新失敗，強制登出
  }
};
```

## 4. API 文件自動化

### 現狀

FastAPI 內建 Swagger UI（`/docs`）和 ReDoc（`/redoc`），但目前：
- 認證需要手動帶入 Bearer Token（不直覺）
- 部分端點缺乏 docstring 和 response_model
- 生產環境建議關閉 /docs（安全考量）

### 改善方案

#### 4-1：為每個路由函式補充 docstring

```python
# 範例：routers/auth.py
@router.post(
    "/exchange-token",
    response_model=TokenResponse,
    summary="Google OAuth Token 交換",
    description="""
    接受 Google ID Token，驗證後：
    1. 建立或更新使用者記錄
    2. 回傳 Bearer Token（即 Google ID Token）
    
    **速率限制**：10 次/分鐘
    """,
    responses={
        200: {"description": "Token 交換成功"},
        401: {"description": "Google Token 無效或已過期"},
        429: {"description": "超過速率限制"},
    },
)
async def exchange_token(...):
    ...
```

#### 4-2：在 Swagger UI 中整合 Bearer Token 認證

```python
# main.py 中加入 security scheme
from fastapi.openapi.utils import get_openapi

def custom_openapi():
    if app.openapi_schema:
        return app.openapi_schema
    openapi_schema = get_openapi(
        title="DataVue API",
        version="1.0.0",
        description="DataVue 數據分析平台 API 文件",
        routes=app.routes,
    )
    # 加入 Bearer Token 安全設定
    openapi_schema["components"]["securitySchemes"] = {
        "BearerAuth": {
            "type": "http",
            "scheme": "bearer",
            "bearerFormat": "Google ID Token",
        }
    }
    openapi_schema["security"] = [{"BearerAuth": []}]
    app.openapi_schema = openapi_schema
    return app.openapi_schema

app.openapi = custom_openapi
```

#### 4-3：生產環境文件保護

```python
# main.py
docs_url = "/docs" if settings.DEBUG_MODE else None
redoc_url = "/redoc" if settings.DEBUG_MODE else None

app = FastAPI(
    title="DataVue API",
    docs_url=docs_url,    # 生產環境隱藏
    redoc_url=redoc_url,  # 生產環境隱藏
)
```

## 5. 前端 TypeScript 遷移

### 現狀

前端目前全為 `.js` / `.jsx` 檔案，`types/api.js` 使用 JSDoc 型別定義。

### 遷移策略：漸進式遷移（不一次性重寫）

#### 第一階段：基礎設定（1 週）

```powershell
cd frontend
# 安裝 TypeScript
npm install -D typescript @types/react @types/react-dom @types/node

# 生成 tsconfig.json（寬鬆模式，允許 JS/TS 混用）
npx tsc --init
```

**tsconfig.json 關鍵設定**：

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM"],
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": false,          // 初期寬鬆，逐步收緊
    "allowJs": true,          // 允許 .js 和 .ts 混用
    "checkJs": false,         // 初期不檢查 .js 檔案
    "noEmit": true,
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

#### 第二階段：高價值模組優先遷移（2–4 週）

遷移優先順序（高 → 低）：

| 模組 | 理由 |
|------|------|
| `src/types/api.js` → `api.ts` | 型別定義轉為真正的 TS interface |
| `src/services/apiClient.js` → `apiClient.ts` | 所有 API 呼叫的核心，型別正確性最重要 |
| `src/hooks/queries/` | Query hooks 的回傳值型別 |
| `src/hooks/mutations/` | Mutation hooks 的參數型別 |
| `src/utils/auth.js` → `auth.ts` | Token 管理邏輯 |

#### 第三階段：元件遷移（4–8 週）

```
src/components/ → 逐一重命名 .jsx → .tsx
src/pages/ → 逐一重命名 .jsx → .tsx
```

#### 型別定義遷移範例

```typescript
// src/types/api.ts（從 JSDoc → TypeScript interface）

// 舊版 JSDoc（api.js）
/**
 * @typedef {Object} User
 * @property {string} id
 * @property {string} email
 * @property {string} name
 */

// 新版 TypeScript（api.ts）
export interface User {
  id: string;
  email: string;
  name: string;
  role: "user" | "admin" | "super_admin";
  status: "active" | "inactive";
  picture?: string;
  created_at: string;
}

export interface AdAccount {
  id: string;
  name: string;
  currency: string;
  status: number;
}

export interface PaginatedResponse<T> {
  items: T[];
  total: number;
  page: number;
  per_page: number;
  has_next: boolean;
}
```
