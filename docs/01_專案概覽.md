# 01_專案概覽 (Project Overview)

## 📌 核心價值：從數字到戰略
**站略 (Site-tegy)** 的存在並非為了堆疊更多圖表，而是為了終結「數據迷航」。
我們核心價值在於：**讓數據不再是冰冷的數字，而是可以執行的戰略。**

## 🎯 目標對象
- **個人品牌**：精準掌控內容影響力與受眾意圖。
- **中小企業**：以有限預算最大化跨平台廣告與搜尋效益。
- **初創團隊**：快速建立數據驅動的決策文化，在競爭中取得先機。

## ⚔️ 解決方案對照
| 傳統數據分析 | 站略 (Site-tegy) 決策 |
| :--- | :--- |
| 分散各平台，整合耗時 | 一站式儀表板，即刻對照 |
| 僅呈現流量與點擊 | 深度意圖分析與行動建議 |
| 決策依賴直覺或零散資訊 | 數據支撐的戰略導向 |
| 擴充功能需大幅改寫代碼 | 插件式模組化架構，快速擴展 |

## 🛠️ 核心功能版圖
1. **Google Search Console (GSC) 洞察**：深入關鍵字戰場，掌握受眾真實意圖。
2. **Facebook Ads 管理**：精準追蹤廣告表現，優化轉化成本。
3. **Google Analytics 4 (GA4) 分析**：全方位的流量路徑分析與內容分組優化。
4. **Site-tegy AI Hub**：與數據深度整合的智慧分析師，提供具備「勝負感」的決策建議。

## 🚀 簡易上手流程
1. **Connect**：完成 Google/Facebook OAuth2 授權。
2. **Analyze**：進入指定模組查看自動彙整的數據視圖。
3. **Strategize**：利用 AI 分析師產出可執行的轉優策略。

## 🏗️ 技術棧 (v2.0.0-modular)
- **基石**：FastAPI (Python) + SQLAlchemy + Pydantic。
- **門面**：React 19 + Vite + Vanilla CSS。
- **大腦**：Google Gemini API + SSE 串流處理。
- **安全**：Fernet 資料加密 + OAuth2 2.0 認證流程。

## 🔮 模組化願景
`v2.0.0-modular` 版本將系統拆解為 **Core (核心)** 與 **Modules (功能模組)**。
這意味著開發者可以像是「插拔插件」一樣，在不影響核心系統的情況下，快速開發或更換如「LINE 數據」或「電子報分析」等新模組。

## 📖 核心術語表
- **Workspace (工作區)**：區分個人與團隊的數據環境。
- **Module (模組)**：獨立的功能套組（如 gsc, fb_ads）。
- **Intent (意圖)**：AI 根據搜尋數據分類出的用戶動機。
- **Strategy (戰略)**：基於數據分析後產出的具體行動指令。
