# GSC Performance Optimization Plan

> 建立日期：2026-01-08  
> 狀態：進行中

提升 GSC 資料載入效能，改善大量資料（25000 筆）的用戶體驗。

---

## 優化項目

### 1. 後端：並行 API 請求
- **檔案**：`backend/gsc_service.py`
- **內容**：將串列 API 請求改為並行（每 5 批次同時發送）
- **效果**：25 次請求 → 5 組並行（**5x 加速**）

### 2. 後端：快取機制
- **檔案**：`backend/gsc_service.py`
- **內容**：新增記憶體快取，5 分鐘內相同查詢直接返回
- **效果**：重複查詢 **0 秒**

### 3. 前端：漸進式載入
- **檔案**：`backend/routers/gsc.py`, `frontend/src/components/GSCStats.jsx`
- **內容**：先顯示首批 1000 筆，背景載入其餘資料
- **效果**：**1-2 秒**即可開始操作

### 4. 前端：虛擬滾動
- **檔案**：`frontend/src/components/GSCStats.jsx`
- **內容**：只渲染可見區域（約 50 列）
- **效果**：25000 筆資料**滾動流暢**
