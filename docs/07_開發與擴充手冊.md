# 07_開發與擴充手冊 (Developer & Extension Guide)

## 🛠️ 開發環境設置
為了確保代碼品質與運行一致性，請遵循以下步驟設置本地開發環境：

### 1. 後端 (FastAPI)
- **環境管理**：建議使用 `venv` 或 `conda`。
- **依賴安裝**：`pip install -r requirements.txt`。
- **環境變數**：複製 `backend/.env.example` 並正確填寫 `GOOGLE_CLIENT_ID` 與 `ENCRYPTION_KEY`。
- **啟動**：`uvicorn main:app --reload`。

### 2. 前端 (React)
- **環境管理**：建議使用 Node.js 18+。
- **安裝與啟動**：`npm install && npm run dev`。
- **環境變數**：確認 `frontend/.env` 中的 `VITE_API_URL` 指向後端網址。

## 🏗️ 模組開發指南
**站略 (Site-tegy)** 是一個模組化驅動的系統。新增功能的標準路徑如下：

### 第一步：建立模組結構
在 `backend/modules/` 下建立您的功能資料夾（例如 `new_service`）：
```
new_service/
├── __init__.py
├── router.py        # 定義 API 端點
├── service.py       # 處理第三方 API 與邏輯
├── dependencies.py  # 模組特定依賴 (如權限檢查)
└── README.md
```

### 第二步：定義權限與 Seed 資料
1. 在 `backend/seeds/permission_seeds.py` 的 `MODULES` 列表新增您的模組 Key。
2. 在 `PERMISSIONS` 字典中定義您的功能細節（如 `new_service:analytics:view`）。
3. 在 `ROLE_PERMISSIONS` 中指派層級。
4. 執行同步腳本：
   ```bash
   python backend/scripts/seed_permissions.py
   ```

### 第三步：註冊路由
在 `backend/main.py` 的 `Router Registration` 區塊引入並包含您的 Router：
```python
from modules.new_service import router as new_service_router
app.include_router(new_service_router, prefix="/api/new-service", tags=["new_service"])
```

## 🛡️ 安全開發規範 (Security Best Practices)
- **Token 加密**：所有存入資料庫的密鑰必須使用 `core.security.encrypt_data` 加密。
- **權限保護**：在 Router 中使用 `require_module("your_module_key")` 進行攔截。
- **異常處理**：應回拋 `AppException` 而非一般的 `Exception`，以確保前端收到格式一致的錯誤訊息。

---

## 🔧 常用維護工具 (Admin Tools)
位於 `backend/scripts/`：
- **`manage_admin.py`**：列出、新增或移除超級管理員。
- **`fix_super_admin.py`**：修復資料庫中損壞的超級管理員權限。
- **`generate_key.py`**：產生用於 `ENCRYPTION_KEY` 的 Fernet 金鑰。

---

## 📝 代碼風格要求
- **Async/Await**：所有 I/O 密集型操作（如資料庫連線、外部 API 請求）必須是非同步的。
- **型別提示 (Type Hinting)**：函數應包含正確的型別提示與 Pydantic 模型驗證。
- **繁體中文註解**：為了維護的一致性，公共 API 或複雜邏輯請包含繁體中文 docstring。
